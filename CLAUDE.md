# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Mosaico is a SwiftUI iOS app (iOS 17+, iPhone only): pick a photo, drag rectangles over sensitive areas, and each area is auto-mosaicked (square pixellate or polygon crystallize); save writes a new photo to the library. Bundle id `info.junz.mosaico`.

## Build & run

The Xcode project is generated by XcodeGen from `project.yml` — never edit `Mosaico.xcodeproj` (git-ignored); edit `project.yml` and regenerate. Info.plist is also generated (`Sources/Info.plist`, git-ignored); its keys live in `project.yml`.

- `just run` — generate, build, install + launch on the iPhone 17 simulator
- `just run-device` — build with `-allowProvisioningUpdates` and install on the first connected iPhone via `devicectl`. Free personal-team signing (`DEVELOPMENT_TEAM` in project.yml): signature expires after 7 days; re-run to refresh.
- `just icon` — regenerate the app icon from `scripts/make_icon.py` (deterministic: fixed seed; needs `uv`)
- `just clean` — remove build products and the generated project

There are no tests or linters configured.

## Verifying changes on the simulator

Drive the UI with idb, not cliclick (cliclick cursor moves but its clicks never land in Simulator):

- `uvx --from fb-idb idb ui tap --udid <UDID> X Y` / `idb ui swipe ... x1 y1 x2 y2 --duration 0.5`
- Coordinates are device points (iPhone 17: 402×874); `xcrun simctl io booted screenshot out.png` captures at 3x
- PhotosPicker taps right after the sheet opens sometimes don't register — wait a few seconds and retry
- Add test photos: `xcrun simctl addmedia booted <file>`

## Architecture

Four Swift files in `Sources/`, all UI state lives in `ImageEditorView`:

- **Non-destructive editing model**: the editor never mutates the picked image. It holds the normalized original plus `rects: [CGRect]` (image-pixel coordinates) and re-renders the display image from scratch on every change via `MosaicProcessor.applyMosaics`. Undo/redo stacks are snapshots of the rect list, not images.
- **Coordinate mapping** (`ImageEditorView`): selection gestures work in view space; `displayScale`/`displayOffset` (aspect-fit `fitScale` × zoom, plus clamped pan) convert between view space and image pixel space. Drag hit-test priority: corner handle (resize) > ×-badge (remove, on tap-sized release) > box interior (move) > empty area (draw new box). Moves are clamped so boxes stay inside the image. The ×-badge is a plain decoration — removal goes through the same hit test, so the badge can't steal the top-right corner handle.
- **Pinch zoom/pan** (`PinchGestureLayer`): SwiftUI has no two-finger gesture reporting centroid movement, so a passthrough `UIPinchGestureRecognizer` is attached to the window (`cancelsTouchesInView = false`); pinches starting outside the editor are ignored, zoom is 1–8× anchored at the moving centroid, and a pinch mid-drag cancels the drag via `dragCancelled`. idb can't synthesize multitouch — test pinch manually (Option-drag in Simulator) or verify math by temporarily forcing `zoom` in state.
- **Orientation invariant** (`UIImage.normalized()` in MosaicProcessor.swift): every picked image is redrawn once so EXIF orientation is baked in and `size` == pixel size. All coordinate math depends on this — don't feed raw picked images to the editor.
- **Core Image pipeline** (`MosaicProcessor`): input is clamped to extent (edge tiles would otherwise average with transparency), filtered whole (CIPixellate or CICrystallize with infinite extent cropped back), then each rect is cropped from the filtered image and composited over the original. Rects use top-left origin; Core Image is bottom-left — the Y-flip happens once when building `ciRect`. The whole-image filtered bitmap is cached (keyed on source/style/tile) so the live preview during a box drag only crops + composites per tick.
- **Editor state reset**: `ContentView` re-ids `ImageEditorView` with a fresh `photoID` per pick, so undo history can never resurface a previously edited photo.

Photo access needs no read permission (PhotosPicker is out-of-process); saving uses add-only access (`NSPhotoLibraryAddUsageDescription`).
